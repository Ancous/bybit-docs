<a id="информация"></a>

Подписка на оповещения ADL и информацию о страховом пуле.

>Зона применения:  
>
>`linear` - контракт (расчет в USDT, USDC, ...)
>
> - `Perpetual` - контракт без даты экспирации
> - `Futures` - контракт с датой экспирации
>
>`inverse` - контракт (расчет в BTC, ETH, ...)
>
> - `Perpetual` - контракт без даты экспирации
> - `Futures` - контракт с датой экспирации

***Частота обновлений***

- 1 с.

***Доступные фильтры***

- `adlAlert.USDT` для бессрочных/поставочных контрактов USDT
- `adlAlert.USDC` для бессрочных/поставочных контрактов USDC
- `adlAlert.inverse` для обратных контрактов.

## Поток данных

<a id="примеры-подписки"></a>

## Примеры подписки

- собственная реализация

  ```python
  import json
  import time
  import websocket
  
  URL = "wss://stream-testnet.bybit.com/v5/public/linear"
  
  def on_message(ws, message):
      print(f"Полученное сообщение: {message}")
  
  def on_error(ws, error):
      print(f"Ошибка: {error}")
  
  def on_close(ws, close_status_code, close_msg):
      print(f"Соединение закрыто: {close_status_code} {close_msg}")
  
  def on_open(ws):
      print("Соединение открыто")
      request = {
          "op": "subscribe",
          "args": [
              "adlAlert.USDT"
          ]
      }
      ws.send(json.dumps(request))
  
  def run_websocket():
      while True:
          ws = websocket.WebSocketApp(URL)
          ws.on_message = on_message
          ws.on_error = on_error
          ws.on_close = on_close
          ws.on_open = on_open
  
          ws.run_forever(
              ping_interval=20,
              ping_timeout=5
          )
  
          print("Соединение потеряно. Попытка переподключения...")
          time.sleep(5)
  
  run_websocket()
  ```

<a id="параметры-получаемых-данных"></a>

## Параметры получаемых данных

|Параметр  |Тип       |Комментарии                                             |
|----------|----------|--------------------------------------------------------|
|topic     |string    |Название потока данных |
|type      |string    |Тип данных. `snapshot` |
|ts        |number    |Метка времени (в миллисекундах), когда система генерирует данные |
|data      |object    |Массив объектов                               |
|c         |string    |Токен пула страхования |
|s         |string    |Название торговой пары |
|b         |string    |Баланс фонда страхования. Используется для определения, запускается ли ADL |
|mb        |string    |Максимальный баланс пула страхования за последние 8 часов |
|i_pr      |string    |Порог соотношения PnL для активации ADL понижения PnL контракта<br><br>- ADL срабатывает, когда коэффициент просадки PnL по символу за последние 8 часов превышает это значение. |
|pr        |string    |Соотношение понижения PnL символа за последние 8 часов. Используется для определения, запускается ли ADL или останавливается |
|adl_tt    |string    |Порог активации для ADL понижения PnL контракта<br><br>- Это условие эффективно только тогда, когда баланс пула страхования больше этого значения; в этом случае понижение за 8 часов, превышающее n%, может запустить ADL |
|adl_sr    |string    |Порог коэффициента остановки для ADL понижения PnL контракта<br><br>- ADL останавливается, когда коэффициент просадки символа за 8 часов падает ниже этого значения. |
