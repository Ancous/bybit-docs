## ***Market***

Раздел **Market** в API Bybit отвечает за получение публичных рыночных данных на всех рынках (спот, деривативы, включая фьючерсы, опционы). Это публичная категория эндпоинтов, не требующая аутентификации, и предназначена для анализа, визуализации и разработки стратегий. Данные обновляются в realtime и включают historical snapshots.

### Основные функции раздела Market

- **Market Data**: Инструменты, тикеры, глубина книги ордеров, последние сделки, candles.
- **Historical Queries**: Архивные данные для backtesting.
- **Risk Metrics**: Funding rates, Greeks (для опционов), delivery records.
- **Real-Time Feeds**: Через REST или WebSocket для live updates.

Это essential для трейдеров и разработчиков ботов.

### Примеры эндпоинтов (данные на основе версии API v5 на момент октябрь 2025 года)

- `GET /v5/market/tickers` — тикеры (цена, volume, change).
- `GET /v5/market/orderbook` — стакан ордеров (глубина до 500).
- `GET /v5/market/time` — серверное время.
- `GET /v5/market/kline` — свечи (candles) с интервалами от 1m до 1M.
- `GET /v5/market/instruments-info` — информация об инструментах (fees, limits).
- ... и другие.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции Market по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Нет аутентификации, но rate limits применяются (IP-based). Для high-frequency используйте WebSocket. Данные historical ограничены (например, kline до 1500 точек).*

## ***Trade***

Раздел **Trade** в API Bybit отвечает за размещение, управление и Cancellation ордеров на всех рынках (спот, linear/perpetual контракты, inverse фьючерсы, опционы). Это приватная категория эндпоинтов для автоматизированной торговли, включая лимитные, рыночные, стоп-ордера и батч-запросы. Раздел обеспечивает высокую скорость исполнения и интеграцию с позициями для risk control.

### Основные функции раздела Trade

- **Order Placement**: Создание одиночных или батч-ордеров (limit, market, TP/SL).
- **Order Management**: Изменение (amend), отмена ордеров.
- **Realtime Tracking**: Историформ и статус ордеров, execution details.
- **Batch Operations**: Массовые действия для усиленияод эффективности.

Это core раздел для ботов и торговых платформ, с поддержкой advanced features как conditional orders.

### Примеры эндпоинтов (данные на основе версии API v5 на момент октябрь 2025 года)

- `POST /v5/order/create` — создание ордера.
- `POST /v5/order/amend` — изменение ордера.
- `POST /v5/order/cancel` — отмена ордера.
- `POST /v5/order/create-batch` — пакетное создание ордеров (до 10).
- `GET /v5/order/realtime` — данные по открытым и закрытым ордеров.
- ... и другие.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции Trade по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Все запросы требуют аутентификации (HMAC-подпись). Лимиты: strict rate limits (IP/account). Для high-frequency trading используйте WebSocket streams.*

## ***Position***

Раздел **Position** в API Bybit отвечает за управление открытыми позициями на деривативах (фьючерсы, опционы, perpetuals), включая просмотр деталей, модификацию (leverage, TP/SL) и закрытие. Это приватная категория эндпоинтов, предназначенная для real-time мониторинга и риск-менеджмента. Раздел интегрируется с трейдингом и помогает оптимизировать стратегии на основе PNL (прибыль/убыток), маржи и греков (для опционов).

### Основные функции раздела Position

- **Position Query**: Список открытых позиций, unrealized PNL, entry price.
- **Modification**: Установка leverage, stop-loss (SL), take-profit (TP).
- **Risk Control**: Проверка margin levels, auto-deleveraging (ADL).
- **Closure**: Частичное или полное закрытие позиций.

Это ключевой раздел для алготрейдеров и risk management.

### Примеры эндпоинтов (данные на основе версии API v5 на момент октябрь 2025 года)

- `GET /v5/position/list` — список позиций по символу/категории.
- `POST /v5/position/set-leverage` — установка кредитное плечо для позиции.
- `POST /v5/position/trading-stop` — настройка TP/SL.
- `GET /v5/position/closed-pnl` — история закрытых позиций и PNL.
- `POST /v5/position/switch-mode` — переключение режима хеджирования (hedge mode).
- ... и другие.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции Position по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Все запросы требуют аутентификации (HMAC-подпись). Лимиты: на основе rate limits Bybit (IP/account). Для Unified Account позиции cross-margin.*

## ***Pre-Upgrade***

Раздел **Pre-Upgrade** в API Bybit отвечает за подготовку, проверку и процесс миграции учетных записей к Unified Trading Account (UTA) — единой модели аккаунта, которая объединяет спот, фьючерсы, опционы и кредитование с sharing equity. Это приватная категория эндпоинтов для пользователей, переходящих с классических типов аккаунтов (Spot, Futures, Derivatives). Раздел помогает избежать проблем при апгрейде, такие как перенос позиций, балансов и комиссий, и проверяет eligibility (подходящих пользователей).

### Основные функции раздела Pre-Upgrade

- **Eligibility Check**: Проверка возможности апгрейда (наличие позиций, балансов).
- **Upgrade Process**: Инициирование и мониторинг апгрейда аккаунта.
- **Data Migration**: Перенос (активов) между аккаунтами (spot -> unified).
- **Historical Data**: История апгрейдов и связанные традиции.

Это важно при переходе к UTA для лучшего risk management и cross-margin trading.

### Примеры эндпоинтов (данные на основе версии API v5 на момент октябрь 2025 года)

- `GET /account/v5/pre-upgrade/execution/list` — сделки до апгрейда.
- `GET /account/v5/pre-upgrade/order/history` — история ордеров до апгрейда.
- `POST /account/v5/pre-upgrade/position/closed` — закрытый PnL по позициям до апгрейда.
- `GET /account/v5/pre-upgrade/delivery-record` — записи поставки/экспирации до апгрейда.
- `GET /account/v5/pre-upgrade/funding/history` — история фондирования до апгрейда.
- `GET /account/v5/pre-upgrade/transaction-log` — журнал транзакций (депозиты/выводы/переводы) до апгрейда.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции Pre-Upgrade по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Все запросы требуют аутентификации (HMAC-подпись). Аппгрейд необратим и может ограничить withdrawal до завершения. Поддерживается только для Main Account, не для Sub-accounts.*

## ***Account***

Раздел **Account** в API Bybit отвечает за управление торговыми счетами пользователя, включая просмотр балансов, позиций, маржи, PNL (прибыль/убыток) и риск-менеджмент. Это приватная категория эндпоинтов, предназначенная для мониторинга и анализа торговой активности на споте, фьючерсах и опционах. Раздел интегрируется с трейдингом и помогает избежать liquidation (ликвидаций) через расчеты маржи.

### Основные функции раздела Account

- **Balances & Wallet**: Текущие балансы (для spot, unified account).
- **Positions**: Открытые позиции, unrealized/realized PNL, leverage.
- **Risk Management**: Маржинальные требования, уровень риска, греки (для опционов).
- **Funding**: Урегулирование фондирования (для фьючерсов).
- **History**: Исторические данные по счету (но основной - в User).

Это ключевой раздел для алготрейдеров и hedge-фондов, позволяющий real-time мониторинг.

### Примеры эндпоинтов (данные на основе версии API v5 на момент октябрь 2025 года)

- `GET /v5/account/wallet-balance` — получить балансы кошелька по типу аккаунта (spot, unified и т.д.).
- `GET /v5/account/fee-rate` — получить текущие торговые комиссии по инструменту/категории.
- `GET /v5/account/collateral-info` — получить список активов-залогов и коэффициенты.
- `GET /v5/account/borrow-history` — история заимствований (для margin trading).
- `GET /v5/account/upgrade-to-uta` — проверить/инициировать апгрейд до Unified Trading Account.
- `GET /v5/account/info` — водная информация об аккаунте/режимах (унифицированный статус и пр.).
- ... и другие.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции Account по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Все запросы требуют аутентификации (HMAC-подпись). Лимиты: основаны на IP и account tier. Для Unified Account доступны cross-margin расчеты.*

## ***Asset***

Раздел **Asset** в API криптовалютной биржи Bybit отвечает за управление активами пользователя. Он предоставляет
эндпоинты для работы с балансами счетов, переводами и другой информацией, связанной с финансами. Это ключевой раздел
для интеграции с кошельками и субаккаунтами в Bybit.

### Основные функции раздела Asset

- **Просмотр баланса и информации об активах**: Получение текущих балансов активов на счету, включая доступные и
  замороженные средства.
- **Переводы между субаккаунтами**: Перемещение средств между основным аккаунтом и субаккаунтами (например,
  для изоляции рисков).
- **Управление субаккаунтами**: Создание, редактирование и просмотр субаккаунтов.
- **Информация о валютах и конверсии**: Получение списка доступных валют и ставок обмена (если применимо).

- `GET /v5/asset/coin-greeks` — получение информации о монетах и греках (для опционов, если включено).
- `GET /v5/asset/transfer/query-asset-info` — запрос информации об активах.
- `POST /v5/asset/transfer/transfer` — перевод средств между аккаунтами.

Для точных деталей и обновлений рекомендую обратиться к официальной документации Bybit API секции Asset по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

## ***User***

Раздел **User** в API Bybit отвечает за управление учетными записями пользователей, включая просмотр и обновление профиля, балансов, транзакций и API-ключей. Это приватная категория эндпоинтов (требует аутентификации через API-ключ и секрет), предназначенная для мониторинга и администрирования личного аккаунта. Раздел поддерживает как спот-торги, так и деривативы (futures/options), с фокусом на безопасность и compliance.

### Основные функции раздела User

- **Account Info**: Просмотр данных пользователя, субаккаунтов, VIP-levels.
- **Balances & Positions**: Обзор балансов (wallet), открытых позиций, PNL (прибыль/убыток).
- **Transaction History**: Журнал депозитов, выводов, торгов, комиссий.
- **API Key Management**: Создание, редактирование, удаление API-ключей (для third-party access).
- **Security**: Настройка прав доступа (permissions), история логинов.

Это необходимо для интеграции с приложением или ботом, где требуется доступ к личным данным.

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания до июня 2024 года)

- `GET /v5/user/balances` — балансы по аккаунту (spot, futures).
- `GET /v5/user/create-api` — создание нового API-ключа.
- `GET /v5/user/query-api` — список API-ключей и их разрешений.
- `POST /v5/user/modify-api` — обновление API-ключа.
- `GET /v5/user/affiliate` — партнерские данные (для referral-программ).
- `GET /v5/user/withdrawal-history` — история выводов.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции User по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Все запросы требуют аутентификации (HMAC-подпись). Лимиты: использование на основе tier-уровня Bybit (VIP). Для субаккаунтов (sub-accounts) доступно ограниченное управление.*

## ***Spread Trading***

Раздел **Spread Trading** в API Bybit отвечает за торговлю спредами (spread), которая представляет собой одновременные ставки на разницу между двумя или более связанными активами (например, контрактами на фьючерсы или парами). Это механизм для хеджирования рисков или спекуляций без прямого влияния на индивидуальные позиции. Подходит для продвинутых трейдеров, желающих воспользоваться корреляциями активов.

### Основные функции раздела Spread Trading

- **Просмотр и управление спредами**: Получение списка доступных спредов, их цен, и параметров (например, спред между двумя perpetual swaps).
- **Размещение и управление ордерами**: Создание, отображение и закрытие позиций в спредах (кросс-ордера на связанные активы).
- **Хеджирование и arbitrage**: Выполнение арбитражных стратегий или нейтрализация экспозиции, когда спред между активами изменяется.
- **Мониторинг позиций**: Просмотр открытых и закрытых позиций по спредам, с деталями прибыли/убытков.

Спреды в Bybit часто включают инструменты как BTC-USDT и ETH-USDT фьючерсы, где трейдер торгует разницей цен.

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания cutoff в июне 2024 года)

- `GET /v5/market/tickers?category=linear` — получение тиковеров, включая спреды (для контекста).
- `POST /v5/order/create` — размещение ордера на спред (указывая связанные символы).
- `GET /v5/position/list` — просмотр позиций, включая спредные.

Для точных деталей и обновлений рекомендую обратиться к официальной документации Bybit API секции Spread Trading по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Функционал может быть ограничен для определенных пользователей (например, профессиональных). API Bybit может обновляться, так что проверьте актуальную версию.*

## ***RFQ Trading***

Раздел **RFQ Trading** (Request for Quote Trading) в API Bybit отвечает за функции, связанные с внебиржевой (over-the-counter, OTC) торговлей криптовалютой на основе запроса котировок. Это предназначено для крупных трейдеров и институциональных клиентов, позволяя запрашивать индивидуальные предложения на сделки без влияния на публичную книгу ордеров (order book). Такой механизм обеспечивает конфиденциальность и кастомизацию сделок с высокими объемами.

### Основные функции раздела RFQ Trading

- **Запрос котировок (RFQ)**: Создание запроса на ценовое предложение от маркет-мейкеров для конкретных активов и объемов.
- **Получение и управление предложениями**: Просмотр, подтверждение или отклонение полученных котировок.
- **Исполнение сделок**: Фиксация сделки на основе согласованной котировки.
- **История и отчеты**: Просмотр истории RFQ-запросов и выполненных сделок.

Это подходит для профессиональной торговли, где стандартный спот- или фьючерс торговли может быть недостаточным из-за ликвидности или влияния на рынок.

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания cutoff в июне 2024 года)

- `POST /v5/rfq/quote` — создание RFQ-запроса (запрос котировки).
- `GET /v5/rfq/quote` — получение списка котировок или деталей.
- `POST /v5/rfq/trade` — подтверждение и исполнение сделки (или аналогичный эндпоинт для завершения).

Для точных деталей и обновлений рекомендую обратиться к официальной документации Bybit API секции RFQ Trading по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Это институциональный функционал, доступный не для всех пользователей. API Bybit может обновляться, так что проверьте актуальную версию.*

## ***Affiliate***

Раздел **Affiliate** в API Bybit отвечает за управление партнерской (реферальной) программой. Он позволяет партнерам (аффилиатам) отслеживать рефералов, комиссии и статистику, генерировать реферальные ссылки и управлять выплатами. Это полезно для тех, кто продвигает Bybit и получает комиссионные за привлечение новых пользователей.

### Основные функции раздела Affiliate

- **Генерация и управление реферальными ссылками**: Создание уникальных ссылок для привлечения новых трейдеров.
- **Просмотр статистики рефералов**: Получение данных о привлеченных пользователях, объемах торговли, комиссиях и т.д.
- **Отчеты по выплатам**: Информация о комиссионных выплатах и балансах реферального счета.
- **Аналитика**: Детали по активности рефералов (например, дата регистрации, ROI).

Это помогает партнерам монетизировать трафик и следить за эффективностью кампаний.

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания cutoff в июне 2024 года)

- `GET /v5/affiliate/info` — получение общей информации о партнерской программе и аккаунте.
- `GET /v5/affiliate/referral-info` — статистика по рефералам (например, количество привлеченных пользователей).
- `GET /v5/affiliate/payout-records` — история выплат комиссий.

Для точных деталей и обновлений рекомендую обратиться к официальной документации Bybit API секции Affiliate по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Чтобы получить доступ к Affiliate API, аккаунт должен быть зарегистрирован в партнерской программе Bybit. API может обновляться, так что проверьте актуальную версию.*

## ***Spot Margin Trade (UTA)***

Раздел **Spot Margin Trade (UTA)** в API Bybit отвечает за маржинальную спот-торговлю в рамках Unified Trading Account (UTA). UTA — это объединенный трейдинговый счет, где средства из спот-кошелька можно использовать для маржинальной торговли. Это позволяет покупать и продавать криптовалютные пары (например, BTC/USDT) с плечом (до 10x), используя заемные средства, чтобы увеличивать прибыль (или риски). Раздел предназначен для управления такими ордерами, позициями и финансированием.

### Основные функции раздела Spot Margin Trade (UTA)

- **Размещение маржинальных ордеров**: Создание, управление и отмена ордеров на спот-рынке с использованием плеча (маржи), включая limit, market и  другие типы ордеров.
- **Управление позициями**: Просмотр открытых позиций по маржинальным спот-парам, включая детали о плече, заемных средствах и PnL.
- **Заимствование и погашение средств**: Borrow (заимствование) и repay (погашение) валюты для увеличения/снижения маржи, с просмотром ставок и лимитов.
- **Финансирование и риски**: Контроль за уровнем маржи, ликвидными средствами и возможной ликвидацией позиций.

Это подходит для трейдеров, желающих спекулировать на спот-рынке с усилением капитала, но с повышенными рисками (возможность ликвидации).

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания cutoff в июне 2024 года)

- `POST /v5/order/create` (с параметром "category": "spot-margin") — размещение ордера на покупку/продажу с маржой.
- `GET /v5/order/history?category=spot-margin` — просмотр истории ордеров.
- `POST /v5/spot-margin-trade/borrow` — заимствование средств для маржи.
- `POST /v5/spot-margin-trade/repay` — погашение долга.
- `GET /v5/position/list?category=spot-margin` — список позиций с маржой.

Для точных деталей и обновлений рекомендую обратиться к официальной документации Bybit API секции Spot Margin Trade (UTA) по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Spot Margin Trade доступен только для Unified Trading Accounts (UTA). Минимальная маржа и комиссии зависят от пары. API может обновляться, так что проверьте актуальную версию.*

## ***Crypto Loan (New)***

Раздел **Crypto Loan (New)** в API Bybit отвечает за управление P2P-кредитованием криптовалюты. Это позволяет пользователям заносить в "Vault" (хранилище) свою крипту для выдачи займов другим трейдерам, получая проценты, или брать взаймы крипту под залог для торговли/инвестиций. Версия "New" указывает на обновленный формат, где займы обрабатываются через оптимизированные механизмы, отдельно от рынка.

### Основные функции раздела Crypto Loan (New)

- **Vault Management**: Размещение активов в Vault для автокредитования или вручную (lending): Указать сумму, процентную ставку и срок. Заработок на процентах от займов.
- **Borrowing**: Получение займа крипты под залог (другой крипты или токенов). Можно использовать для маржинальной торговли без риска ликвидации.
- **Loan Management**: Просмотр активных займов, погашение долга, досрочное завершение. Поддержка гибких условий (fixed или flexible rates).
- **Аналитика**: Статистика доходов, ставок, ликвидности и рисков (включая APR).

Это полезно для пассивного дохода (lending) или финансирования трейдинга (borrowing), с поддержкой популярных токенов как BTC, USDT, ETH.

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания cutoff в июне 2024 года)

- `GET /v5/crypto-loan/vault/asset-info` — информация по доступным активам для Vault.
- `POST /v5/crypto-loan/vault/deposit` — внесение средств в Vault для lending.
- `POST /v5/crypto-loan/borrow` — запрос займа крипты.
- `GET /v5/crypto-loan/loan-history` — история ваших займов.
- `POST /v5/crypto-loan/repay` — погашение займа.

Для точных деталей и обновлений рекомендую обратиться к официальной документации Bybit API секции Crypto Loan (New) по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Crypto Loan требует верификации аккаунта. Ставки зависят от рынка и могут изменяться. API подчеркивает, что lending несет риски, включая волатильность активов в Vault.*

## ***Institutional Loan***

Раздел **Institutional Loan** в API Bybit предназначен для институционального кредитования, где крупные клиенты (такие как хедж-фонды, OTC-трейдеры) могут брать займы в крупных объемах под залог криптовалюты. Это часть эксклюзивных услуг для verified институциональных аккаунтов, позволяющая получать финансирование для торговли, хеджинга или инвестиций с гибкими условиями, такими как более низкие ставки и персонализированные сроки. Раздел фокусируется на безопасных и конфиденциальных транзакциях, часто интегрированных с OTC-торговлей.

### Основные функции раздела Institutional Loan

- **Borrowing**: Заимствование больших сумм (миллионы долларов) крипты или фиата под залог активов. Поддержка нескольких валют (BTC, ETH, USDT и др.) с custome-условиями (ставки, сроки).
- **Loan Management**: Просмотр и управление активными займами, включая расчет процентов, collateral leverage ratio и погашение.
- **Collateral Management**: Анализ залога, LTV (loan-to-value ratio) и риски автоматической ликвидации при падении цены залога.
- **Интеграция с другими сервисами**: Совмещение с Institutional Trading для seamless опыта, с VIP-поддержкой.

Это подходит для профессиональных трейдеров, нуждающихся в капитале без волатильности обычного рынка, но требует одобрения от Bybit и высокой верификации (KYC level 4+).

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания до июня 2024 года)

- `GET /v5/institution-asset/vault/borrow-info` — информация по доступным займам и ставкам.
- `POST /v5/institution-asset/vault/borrow` — запрос займа.
- `GET /v5/institution-asset/loan/loan-history` — история institutional loans.
- `POST /v5/institution-asset/vault/repay` — погашение долга.
- `GET /v5/institution-asset/vault/collateral-info` — детали по залогу.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции Institutional Loan по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Этот раздел доступен только для институциональных аккаунтов с высоким уровнем верификации. Минимальные суммы и ставки обсуждаются индивидуально. API может требовать специальных разрешений и поддерживает OTC-трейдинг.*

## ***Exchange Broker***

Раздел **Exchange Broker** в API Bybit отвечает за управление брокерскими операциями в рамках программы Bybit Broker (Affiliate или Broker Program), где брокеры (трейдеры с реферальным статусом) могут создавать суб-аккаунты, управлять комиссиями и рефералами, получать бонусы и отчеты. Это API для автоматизации процессов White Label: брокеры под своим брендом предлагают клиентам трейдинг на Bybit, зарабатывая процент от комиссий клиентов. Раздел ориентирован на маркетмейкеров и агентов, облегчая массовое управление.

### Основные функции раздела Exchange Broker

- **Sub-account Management**: Создание и управление суб-аккаунтами (для клиентов брокера), настройка прав доступа, балансов и комиссий.
- **Commission & Rebate Management**: Настройка и просмотр реферальных платежей (rebates): Брокеры получают процент от объемов торговли своих рефералов (например, 10-30% от maker/taker fee).
- **Reporting & Analytics**: Доступ к отчетам по доходам, объемам, статистике рефералов, включая API для расчетов и выплат.
- **Integration**: Интеграция с другими API для seamless-торговли, включая spot/futures рынки.

Это помогает брокерам масштабировать бизнес, привлекать клиентов и мониторить доходы без ручной работы.

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания до июня 2024 года)

- `GET /v5/broker/exchange/info` — информация по брокерскому статусу и настройкам.
- `POST /v5/broker/exchange/sub-create` — создание суб-аккаунта.
- `GET /v5/broker/exchange/rebate-info` — детали по rebates и комиссиям.
- `GET /v5/broker/exchange/volume` — отчеты по торговым объемам рефералов.
- `POST /v5/broker/exchange/rebate-modify` — изменение ставок rebates.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции Exchange Broker по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Доступ к этому разделу требует верификации брокерского аккаунта. Минимальные объемы для выплат зависят от статуса. API может требовать специальных прав и поддерживает multi-level affiliate структуры.*

## ***Earn***

Раздел **Earn** в API Bybit отвечает за управление продуктами доходности (staking и сбережения), где пользователи могут инвестировать криптовалюту в фиксированные программы с APY (annual percentage yield) для пассивного заработка. Это включает Savings Account (гибкие депозиты) и Dual Investment (структурированные продукты), позволяющие зарабатывать на активах без активной торговли. Раздел поддерживает различные активы (BTC, ETH, USDT и др.), сроки и риски, с фокусом на институциональных или VIP-клиентах для лучших ставок.

### Основные функции раздела Earn

- **Product Discovery**: Просмотр доступных Earn-продуктов, включая ставки APY, минимальные суммы, durations и риски.
- **Subscription & Management**: Инвестирование средств в продукт, мониторинг позиций и доходов.
- **Payouts & Withdrawals**: Автоматический расчет процентов, запрос на вывод средств.
- **Analytics**: Отчеты по прибыльности, истории транзакций.

Это подходит для консервативных инвесторов, желающих диверсифицировать портфель с низким риском (за исключением Dual Investment с leverage).

### Примеры эндпоинтов (на основе версии API v5, актуальной на момент моего знания до июня 2024 года)

- `GET /v5/earn/products` — список доступных Earn-продуктов.
- `POST /v5/earn/subscribe` — подписка на продукт (инвестирование).
- `GET /v5/earn/positions` — позиции пользователя и доходы.
- `POST /v5/earn/unsubscribe` — вывод средств из продукта.
- `GET /v5/earn/rate-history` — история ставок.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции Earn по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: Продукты Earn могут иметь блокировки (lock periods). Возможны изменения условий; всегда проверяйте риски. Для институциональных клиентов доступны customized варианты.*

## ***WebSocket Stream***

Раздел **WebSocket Stream** в API Bybit отвечает за потоковую передачу данных в реальном времени (real-time) через протокол WebSocket. Это альтернатива REST API для получения быстрых обновлений без постоянных опросов сервера, таких как рыночные данные, ордеры, позиции и исполнение сделок. Раздел поддерживает как публичные каналы (market data) для всех пользователей, так и приватные (私人) для аутентифицированных аккаунтов. Связь устанавливается через WebSocket-подключение, с поддержкой ping/pong для поддержания соединения.

### Основные функции раздела WebSocket Stream

- **Market Data**: Реал-тайм данные о торгах, тикерах, ордербуках.
- **User Data**: Обновления по ордерам, позициям, балансам (требует аутентификации).
- **High Frequency**: Оптимизирован для частых обновлений, идеален для алготрейдинга, ботов или приложений.
- **Customization**: Фильтры по торговым инструментам (символам, парам), типам данных.

Максимальное соединение — одно per user, но каналы асинхронны.

### Примеры топиков и каналов (на основе версии API v5, актуальной на момент моего знания до июня 2024 года)

- **Публичные (Public) каналы** (подключение: `wss://stream.bybit.com/v5/public/{topic}`):
  - `ticker` — реал-тайм тикеры (цены, объемы).
  - `trade` — последние сделки.
  - `orderbook` — обновления ордербука (глубина: 1, 10, 25, 500).
  - `kline` — свечи (candlesticks) для графиков.
- **Приватные (Private) каналы** (подключение: `wss://stream.bybit.com/v5/private`):
  - `order` — обновления статуса ордеров.
  - `execution` — исполнение сделок.
  - `position` — изменения позиций (spot/futures).
  - `wallet` — обновления баланса.

Примеры сообщений: Аутентификация для private (с HMAC-подписью), подписка на каналы через JSON-запросы.

### Примеры эндпоинтов или интеграции

- Нет жестких REST-эндпоинтов; подключение через WebSocket URL.
- Для примера кода: Используйте библиотеки вроде `websocket-client` в Python для подписки на канал.

Для точных деталей рекомендую обратиться к официальной документации Bybit API секции WebSocket Streams по [адресу](https://bybit-exchange.github.io/docs/v5/intro).

*Примечание: WebSocket требует стабильного интернет-соединения. Лимиты: до 50 каналов на соединение. Для аутентифицированных данных нужен API Key. Поддерживает сжатие данных.*

## ***Rate Limit***

Раздел **Rate Limit** в API Bybit описывает политику ограничений на частоту запросов для защиты от злоупотреблений и обеспечения стабильности платформы. Это не отдельная категория эндпоинтов, а общие правила, применяемые ко всем API-вызовам (public и private). Ограничения зависят от аккаунта/IP и категории эндпоинтов.

### Основные функции раздела Rate Limit

- **Request Quotas**: Лимиты на вызовы (в запросах/мин, час) для эндпоинтов Market, Trade, Position и т.д.
- **Categories**: Разные лимиты для public (выше) и private (строже, с аккаунт-зависимыми bounds).
- **Tracking**: Через ответные headers (X-RateLimit*, остаток limits).
- **Penalties**: Exceed limits — временный бан, throttling или block.

Примеры limits (на основе v5 docs, актуально до июня 2024)

- Public (Market): 200 req/min per endpoint.
- Private (Trade): 50 req/min, stricter for high-frequency.
- Batch orders: +30 req/min klock.

Во избежание проблем следите за headers или используйте WebSocket.

### Примеры лимитов (не эндпоинты, но политика)

- Position/Private: 100 req înche/min, penalize за abuse.
- Общие: IP-blacklist при спаме; account lock за торговлю сверх лимита.

Для точных limits и updates рекомендую официальную документацию Bybit API по [адресу](https://bybit-exchange.github.io/docs/v5/intro) секцию Rate Limits.

*Примечание: Лимиты динамичны, зависят от бизнес-веса. WebSocket имеет свои rules. Проверяйте docs перед интеграцией.*
